# CleaniSense — Backend API Documentation

This document serves as the single reference for frontend developers integrating with the CleaniSense backend.

---

## 1. Authentication & Global Headers

CleaniSense uses **Firebase Authentication** for user management.

### Headers required for authenticated endpoints:
```http
Authorization: Bearer <Firebase_ID_Token>
X-Request-ID: <UUID> (Optional correlation ID; will be generated by backend if missing)
```

- **Bearer Token**: The Firebase client-side ID token (JWT) generated by `user.getIdToken()`.
- **Response ID Header**: The backend returns the `X-Request-ID` header in all responses for tracing.

---

## 2. Standard Response Envelopes

### 2.1 Success Response Envelope
All successful requests return a status code of `200` or `201` with the following structure:
```json
{
  "success": true,
  "message": "Action completed successfully",
  "data": { ... } // Single object, list, or null
}
```

### 2.2 Error Response Envelope
All failed requests (validations, unauthorized, exceptions) return standard HTTP error codes with this envelope:
```json
{
  "success": false,
  "message": "Error description message text",
  "error": {
    "code": "VALIDATION_ERROR | NOT_FOUND | UNAUTHORIZED | FORBIDDEN | INTERNAL_ERROR",
    "details": [ // Optional field-specific details (e.g. on validation error)
      {
        "field": "title",
        "message": "Field required"
      }
    ]
  }
}
```

## 3. Endpoints reference

### 3.1 Authentication

#### `POST /api/v1/auth/login`
- **Description**: Verifies a client-side Firebase ID token, creates the user profile if it is their first login, and registers the session.
- **Request Body**:
  ```json
  {
    "id_token": "string"
  }
  ```
- **Response Data (`data` field)**:
  ```json
  {
    "id": "uuid",
    "email": "string",
    "name": "string",
    "profile_picture": "string (url)",
    "role": "citizen | municipality_officer | municipality_admin | super_admin",
    "is_active": true
  }
  ```

---

### 3.2 User Profile & Preferences

#### `GET /api/v1/profile`
- **Description**: Retrieves current logged-in citizen profile and preferences.
- **Response Data (`data` field)**:
  ```json
  {
    "id": "uuid",
    "email": "string",
    "name": "string",
    "profile_picture": "string (url)",
    "role": "citizen",
    "is_active": true,
    "preferences": {
      "language": "en | hi | gu | bn | ta | te",
      "theme": "system | light | dark",
      "notifications_enabled": true
    }
  }
  ```

#### `PUT /api/v1/profile`
- **Description**: Updates profile display name and profile picture URL.
- **Request Body**:
  ```json
  {
    "name": "string (max 100)",
    "profile_picture": "string (url, optional)"
  }
  ```
- **Response Data (`data` field)**: Same structure as `GET /api/v1/profile`.

#### `GET /api/v1/profile/preferences`
- **Description**: Retrieves specific user configuration preferences.
- **Response Data (`data` field)`**:
  ```json
  {
    "language": "string",
    "theme": "string",
    "notifications_enabled": true
  }
  ```

#### `PUT /api/v1/profile/preferences`
- **Description**: Updates user theme, language locale, and notifications settings.
- **Request Body**:
  ```json
  {
    "language": "en | hi | gu | bn | ta | te (optional)",
    "theme": "system | light | dark (optional)",
    "notifications_enabled": bool (optional)
  }
  ```
- **Response Data (`data` field)**: Same structure as `GET /api/v1/profile/preferences`.

- **Response Data (`data` field)**:
  ```json

      "total_reports": 5,
      "active_reports": 4,
      "resolved_reports": 1,
      "nearby_hotspots": 2
    },
    "recent_reports": [
      {
        "id": "uuid",
        "title": "string",
        "status": "submitted | ai_validation_completed | ...",
        "created_at": "datetime"
      }
    ],
    "nearby_hotspots": [
      {
        "id": "uuid",
        "title": "string",
        "latitude": 23.03,

    ],
    "unread_notifications": 3,
    "preferences": {
      "language": "en",
      "theme": "system",
      "notifications_enabled": true
  ```

#### `GET /api/v1/dashboard/config`

- **Response Data (`data` field)**:

### 3.4 Complaints Lifecycle

#### `POST /api/v1/complaints`
- **Description**: Creates a new environmental complaint report.
- **Request Body**:
  ```json
  {
    "title": "string (min 5, max 100)",
    "description": "string (min 10, max 1000)",
    "category_id": "uuid",
    "location_name": "string",
    "latitude": 23.03,
    "longitude": 72.50,
    "municipality_id": "uuid (optional)"
  }
  ```
- **Response Data (`data` field)**:
  ```json
  {
    "id": "uuid",
    "title": "string",
    "status": "submitted",
    "created_at": "datetime"
  }
  ```

#### `GET /api/v1/complaints`
- **Description**: Returns paginated list of active complaints submitted by the user.
- **Query Parameters**:
  - `page`: `int` (default 1)
  - `page_size`: `int` (default 20, max 100)
- **Response Data (`data` field)**:
  ```json
  {
    "items": [
      {
        "id": "uuid",
        "title": "string",
        "status": "string",
        "created_at": "datetime"
      }
    ],
    "total": 12,
    "page": 1,
    "page_size": 20,
    "has_next": false
  }
  ```

#### `GET /api/v1/complaints/history`
- **Description**: Retrieves advanced filtered, sorted, and searched complaint history records.
- **Query Parameters**:
  - `status`: `string` (optional, draft | submitted | in_progress | resolved, etc.)
  - `category_id`: `uuid` (optional)
  - `search`: `string` (optional)
  - `sort_by`: `string` (default `created_at`)
  - `sort_order`: `string` (default `desc`)
  - `page`: `int` (default 1)
  - `page_size`: `int` (default 20)
- **Response Data (`data` field)**: Same paginated list structure as `GET /api/v1/complaints`.

#### `GET /api/v1/complaints/{id}`
- **Description**: Retrieves full details of a specific complaint, including nested categories, municipality details, status history timeline, attachments, and resolution report.
- **Response Data (`data` field)**:
  ```json
  {
    "id": "uuid",
    "title": "string",
    "description": "string",
    "status": "submitted | resolved | ...",
    "location_name": "string",
    "latitude": 23.03,
    "longitude": 72.50,
    "created_at": "datetime",
    "category": {
      "id": "uuid",
      "name": "string",
      "icon": "string",
      "color": "string"
    },
    "municipality": {
      "id": "uuid",
      "name": "string",
      "district": "string",
      "state": "string"
    },
    "timeline": [
      {
        "id": "uuid",
        "status": "submitted",
        "remarks": "string",
        "created_at": "datetime"
      }
    ],
    "attachments": [
      {
        "id": "uuid",
        "public_url": "string (url)",
        "file_type": "image | document",
        "file_name": "string",
        "file_size_bytes": 1024
      }
    ],
    "resolution": {
      "id": "uuid",
      "summary": "string",
      "department": "string",
      "officer_name": "string",
      "actions": "string",
      "before_image_url": "string (url)",
      "after_image_url": "string (url)",
      "completed_at": "datetime"
    }
  }
  ```

#### `PUT /api/v1/complaints/{id}`
- **Description**: Updates complaint details. Allowed only if the complaint is in `draft` or `submitted` status.
- **Request Body**:
  ```json
  {
    "title": "string (optional)",
    "description": "string (optional)",
    "category_id": "uuid (optional)",
    "location_name": "string (optional)",
    "latitude": "float (optional)",
    "longitude": "float (optional)"
  }
  ```
- **Response Data (`data` field)**: Returns basic complaint response object (title, status, id, created_at).

#### `DELETE /api/v1/complaints/{id}`
- **Description**: Soft-deletes a complaint. Allowed only if status is `draft` or `submitted`.
- **Response Data (`data` field)**:
  ```json
  {
    "id": "uuid"
  }
  ```

#### `POST /api/v1/complaints/{id}/attachments`
- **Description**: Uploads a single media attachment file (JPEG, PNG, or PDF; max size 10MB) to the complaint. Limit of 5 attachments total per complaint.
- **Request Header**: `Content-Type: multipart/form-data`
- **Form Data**:
  - `file`: `UploadFile` (binary data)
- **Response Data (`data` field)**:
  ```json
  {
    "id": "uuid",
    "public_url": "string (url)",
    "file_type": "image | document",
    "file_name": "string",
    "file_size_bytes": 450123
  }
  ```

#### `GET /api/v1/complaints/{id}/resolution`
- **Description**: Retrieves the official municipality resolution details for a resolved complaint.
- **Response Data (`data` field)**: Same structure as the nested `resolution` object in `GET /api/v1/complaints/{id}` details response.

### 3.5 Notifications

#### `GET /api/v1/notifications`
- **Description**: Retrieves a paginated list of notifications for the user.
- **Query Parameters**:
  - `is_read`: `bool` (optional, filter by status)
  - `page`: `int` (default 1)
  - `page_size`: `int` (default 20)
- **Response Data (`data` field)**:
  ```json
  {
    "items": [
      {
        "id": "uuid",
        "complaint_id": "uuid (optional)",
        "title": "string",
        "message": "string",
        "type": "submitted | accepted | resolved | ...",
        "is_read": false,
        "created_at": "datetime"
      }
    ],
    "total": 3,
    "page": 1,
    "page_size": 20,
    "has_next": false
  }
  ```

#### `GET /api/v1/notifications/unread-count`
- **Description**: Returns the count of unread notifications for the user.
- **Response Data (`data` field)**:
  ```json
  {
    "count": 3
  }
  ```

#### `PUT /api/v1/notifications/read-all`
- **Description**: Marks all notifications belonging to the user as read.
- **Response Data (`data` field)**:
  ```json
  {
    "count_marked": 3
  }
  ```

#### `PUT /api/v1/notifications/{id}/read`
- **Description**: Marks a specific notification as read.
- **Response Data (`data` field)**: Returns the single notification response object with `"is_read": true`.

### 3.6 Environmental Hotspots

#### `GET /api/v1/hotspots`
- **Description**: Returns a list of environmental pollution hotspots.
- **Query Parameters**:
  - `latitude`: `float` (optional, search center)
  - `longitude`: `float` (optional, search center)
  - `radius_km`: `float` (default `5.0`, ge=0.1, le=100.0)
  - `severity`: `high | medium | low` (optional)
- **Response Data (`data` field)**:
  ```json
  [
    {
      "id": "uuid",
      "title": "string",
      "description": "string",
      "latitude": 23.028,
      "longitude": 72.505,
      "severity": "high",
      "is_active": true
    }
  ]
  ```

#### `GET /api/v1/hotspots/{id}`
- **Description**: Retrieves single hotspot details.
- **Response Data (`data` field)**: Same structure as single hotspot object in list above.
